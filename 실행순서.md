이제 모든 '코드 헌법'과 '매뉴얼'이 완벽하게 준비되었습니다. 클로드 코드(Claude Code)를 사용하여 프로젝트를 시작할 때는, 우리가 정의한 **`workflow-manual.md`**의 표준 작업 절차(SOP)에 따라 진행하시면 됩니다.

시니어 개발자로서 클로드에게 업무를 위임하는 **4단계 실전 절차**를 안내해 드립니다.

---

### 1단계: 클로드 소환 및 프로젝트 분석
가장 먼저 터미널에서 프로젝트 폴더로 이동한 뒤 클로드를 실행하고, 전체적인 맥락을 파악하게 합니다.

1.  **명령어 입력:** `claude`
2.  **첫 인사 및 탐색:** `"안녕, 이 프로젝트의 모든 가이드라인 문서를 읽고 현재 폴더 구조를 분석해줘."`
    * 클로드가 **`CLAUDE.md`**를 읽고 프로젝트의 기술 스택(Java 21, Spring Boot 3.4)과 아키텍처 원칙을 인지하게 됩니다.

### 2단계: 전략 및 기획 (매뉴얼 [A])
새로운 기능을 만들 때는 반드시 **[A] 단계**부터 시작하여 '계획(Plan)'을 먼저 확정해야 합니다.

1.  **명령어:** `"새로운 '사용자 관리 서비스'를 만들 거야. **매뉴얼 [A]**로 진행해줘."`
2.  **클로드의 동작:** * **[Planner]**와 **[Lead Architect]** 페르소나로 전환합니다.
    * `architecture.md`에 정의된 MSA 및 DDD 4계층 구조에 따라 어떤 프로젝트를 나누고 어떤 파일을 생성할지 계획을 세웁니다.
    * 구현 전 계획을 제시하고 사용자(당신)의 컨펌을 기다립니다.

### 3단계: 구현 및 실행 (매뉴얼 [B])
계획이 맘에 든다면, 실제 코드를 작성하게 합니다.

1.  **명령어:** `"계획이 좋아. 이제 **매뉴얼 [B]**로 실제 구현을 시작해."`
2.  **클로드의 동작:** * **[Senior Software Engineer]** 페르소나로서 `coding-standards.md`를 준수하며 코딩합니다.
    * Java 21의 가상 스레드를 활용하고, DTO는 `record` 타입을 사용하며 생성자 주입 방식을 적용합니다.
    * `infrastructure.md`에 정의된 표준 Dockerfile 형식을 사용하여 배포 환경까지 고려합니다.

### 4단계: 검증 및 마감 (매뉴얼 [B])
코드가 완성되면 품질을 검증하고 마무리합니다.

1.  **명령어:** `"수고했어. 이제 **매뉴얼 [C]**로 검증하고 마무리하자."`
2.  **클로드의 동작:** * **[QA Auditor]**로서 JUnit 5 단위 테스트를 작성하고 엣지 케이스를 점검합니다.
    * 빌드 에러가 발생하면 스스로 3회까지 수정을 시도합니다.
    * 작업이 완료되면 `git-convention.md`에 따라 `feature` 브랜치에 커밋을 생성합니다.

### 💡 숙련된 사용자를 위한 팁

* **대화 초기화:** 한 기능의 구현이 완전히 끝났다면 **`/clear`**를 입력하여 이전 대화 맥락을 지워주세요. 그래야 다음 작업 시 불필요한 토큰 소모를 막아 **할당량(Quota)**을 아낄 수 있습니다.
* **모델 전환:** 단순한 문서 수정이나 파일명 변경은 **`/model haiku`**를, 복잡한 설계는 **`/model default`**를 사용하여 효율적으로 작업하세요.
* **결과 검증:** 클로드가 작업을 마친 뒤에는 반드시 여러분이 코드를 직접 훑어보며 `ai-collaboration.md`의 원칙들이 잘 지켜졌는지 확인하는 습관을 들이는 것이 좋습니다.

**Claude Code 세션 내부**에서 조작할 때는 말씀하신 대로 **슬래시(/) 명령어**를 사용하는 것이 정확합니다.
PowerShell 사용 환경에 맞춰, 다시 정확하게 정리해 드릴게요.

---

## 1. 세션 내부 명령 (실행 중일 때)

Claude Code가 실행 중인 상태에서 터미널에 직접 입력하세요.

* **현재 모델 확인 및 변경:** `/model`
    * `/model`만 입력: 현재 사용 중인 모델 이름이 표시됩니다.
    * `/model <모델명>`: 세션 중간에 모델을 즉시 변경합니다. (단, Anthropic API 내의 모델끼리만 가능)
* **사용량(토큰/비용) 확인:** `/stats`
    * 현재 대화에서 사용된 인풋/아웃풋 토큰과 누적 비용을 보여줍니다.
* **설정 확인:** `/config`
    * 현재 적용된 전체 설정을 확인합니다.

---

## 2. Ollama(오픈소스)로 교체하는 법 (상세 절차)

진행 중에 사용량 초과로 멈췄을 때, **세션 내부에서 `/model` 명령으로 Ollama 모델을 바로 불러오는 것은 불가능**합니다. 그 이유는 Claude Code가 접속하는 '서버 주소(Base URL)' 자체가 Anthropic에서 여러분의 로컬 PC(Ollama)로 바뀌어야 하기 때문입니다.

따라서 아래 순서대로 진행해야 합니다.

### Step 1: 현재 세션 종료
* `/exit`를 입력하여 Claude Code를 종료합니다.

### Step 2: PowerShell 환경 변수 설정
Ollama를 Anthropic 서버인 것처럼 인식하게 만드는 과정입니다. PowerShell 창에 다음을 입력하세요.
```powershell
$env:ANTHROPIC_BASE_URL = "http://localhost:11434/v1"
$env:ANTHROPIC_API_KEY = "ollama"
```

### Step 3: Ollama 모델로 재실행
이제 `deepseek-r1:8b` 모델을 지정해서 Claude Code를 다시 켭니다.
```powershell
claude --model deepseek-r1:8b
```
*이렇게 하면 Claude Code의 인터페이스는 그대로 쓰면서, 머리(LLM)만 내 컴퓨터의 DeepSeek으로 바뀌어 동작합니다.*

---

## 3. 요약 가이드

| 목적 | 명령어 (PowerShell/Claude 내부) |
| :--- | :--- |
| **현재 모델 확인** | `/model` |
| **현재 세션 사용량 확인** | `/stats` |
| **사용량 초과 시 대처** | 세션 종료 → 환경변수 설정 → `claude --model <로컬모델>` |
| **다시 Claude로 복귀** | PowerShell 창 끄고 새로 열기 (환경변수 초기화) |

**추가 팁:** `deepseek-r1:8b`는 추론 능력이 좋지만 코딩 도구 호출(Tool Use) 능력이 가끔 불안정할 수 있습니다. 만약 "도구를 사용할 수 없다"는 에러가 반복되면, 코딩 전문 모델인 `qwen2.5-coder:7b`를 다운로드해서 똑같은 방식으로 시도해 보세요. 훨씬 매끄럽게 작동할 겁니다.
